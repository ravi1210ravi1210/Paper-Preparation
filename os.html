  1. Define Operating System.
    An Operating System (OS) is system software that manages hardware and software resources and provides services to applications.
  2. Main functions of OS?
    Process management, memory management, file management, device management, security.
  3. What is a process?
    A process is a program in execution.
  4. Define thread.
    A thread is the smallest unit of CPU execution within a process.
  5. What is CPU scheduling?
    It determines which process runs next on the CPU.
  6. What is deadlock?
    A situation where two or more processes are waiting for each other forever.
  7. What is paging?
    A memory management scheme that eliminates the need for contiguous allocation of physical memory.
  8. Define semaphore.
    A synchronization tool used to control access to shared resources.
  9. What is a system call?
    An interface between user programs and the OS kernel.
  10. What is virtual memory?
    A technique that allows execution of processes not completely in memory using secondary storage.

Q1. Explain the main functions of an Operating System.

  • Process Management: Manages processes, CPU scheduling, and multitasking.
  • Memory Management: Allocates and deallocates memory space.
  • File Management: Organizes data into files and directories.
  • Device Management: Controls I/O devices using drivers.
  • Security and Protection: Prevents unauthorized access.

Q3. Explain CPU Scheduling Algorithms.

  • FCFS (First Come First Serve): Processes are executed in arrival order.
  • SJF (Shortest Job First): Process with shortest burst time executes first.
  • Priority Scheduling: Highest priority process executes first.
  • Round Robin: Each process gets a fixed time slice (quantum).

Q4. Explain Deadlock with conditions and prevention methods.

Deadlock: Occurs when processes hold resources and wait for others.

  • Conditions for Deadlock:
    1. Mutual Exclusion
    2. Hold and Wait
    3. No Preemption
    4. Circular Wait
  • Prevention Methods:
    1. Avoid circular waiting (ordering of resources)
    2. Allow preemption
    3. Use resource allocation graph

Q5. Explain Paging and Segmentation.

  • Paging: Divides physical memory into fixed-size blocks (pages).
  • Segmentation: Divides memory based on logical divisions (code, data, stack).

Q6. Explain File Management System.

It deals with creation, deletion, reading, and writing of files. The OS maintains file attributes like name, size, and type.

Q10. Explain the concept of System Calls.

System calls allow user-level programs to interact with the kernel (e.g., file creation, I/O).
