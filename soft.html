Q3. Explain the phases of SDLC in detail.

  1. Requirement Analysis: Understanding what the user needs.
  2. Design: Converting requirements into a system model.
  3. Coding: Actual implementation using programming languages.
  4. Testing: Detecting and fixing defects.
  5. Maintenance: Enhancing and fixing issues post-deployment.

Q5. Explain Software Testing and its types.

  • White Box Testing: Testing internal logic and structure.
  • Black Box Testing: Testing functionality without knowing code.
  • Unit Testing: Testing individual modules.
  • Integration Testing: Testing combined modules.
  • System Testing: Testing the whole system.

Q8. Explain Software Maintenance and its types.

  • Perfective: Improving performance or features.
